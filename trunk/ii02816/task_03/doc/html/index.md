#Документация проекта "PID Controller & System Model"
##Файлы
###main.cpp
**Основная программа для демонстрации работы ПИД-регулятора и модели системы**
Программа тестирует взаимодействие ПИД-регулятора с различными моделями объекта управления и выводит результаты в консоль.

**main**
Основная функция программы:

Создает ПИД-регулятор с заданными коэффициентами

Создает модель процесса

Запускает simulation системы

Выводит результаты на каждом шаге

###pid_controller.h / pid_controller.cpp
Заголовочный файл и реализация класса ПИД-регулятора
Класс, реализующий дискретный ПИД-регулятор (пропорционально-интегрально-дифференциальный регулятор).

####Класс PIDController
**Атрибуты:**

`kp_` — Пропорциональный коэффициент (Kp)

`ki_` — Интегральный коэффициент (Ki)

`kd_` — Дифференциальный коэффициент (Kd)

`dt_`— Время дискретизации (шаг simulation)

`integral_` — Накопленная интегральная составляющая

`prev_error_` — Ошибка на предыдущем шаге (для дифференциальной составляющей)

`output_limit_` — Ограничение выходного сигнала (опционально)

**Методы:**

`PIDController(double kp, double ki, double kd, double dt)`
Конструктор ПИД-регулятора

`kp` — пропорциональный коэффициент

`ki` — интегральный коэффициент

`kd` — дифференциальный коэффициент

`dt` — время дискретизации

`double calculate(double setpoint, double current_value)`
Вычисляет управляющее воздействие на основе текущей ошибки

`setpoint` — заданное значение (целевое)

`current_value` — текущее измеренное значение

**Возвращает:** управляющее воздействие u(k)

`void reset()`
Сбрасывает внутреннее состояние регулятора (интеграл и предыдущую ошибку)

`void setOutputLimits(double min, double max)`
Устанавливает ограничения на выходное значение регулятора

`min` — минимальное допустимое значение

`max` — максимальное допустимое значение

`std::vector<double> getParameters() const`
Возвращает текущие параметры регулятора

**Возвращает:** вектор [kp, ki, kd, dt]

###model.h / model.cpp
**Заголовочный файл и реализация модели объекта управления**
Класс, моделирующий динамику объекта управления, который может быть использован для тестирования ПИД-регулятора.

####Класс SystemModel
Атрибуты:

`a_` — Коэффициент системы (параметр модели)

`b_` — Коэффициент системы (параметр модели)

`prev_output_` — Предыдущее выходное значение системы

`time_constant_` — Постоянная времени системы

`noise_level_` — Уровень шума измерений (опционально)

**Методы:**

`SystemModel(double a, double b, double initial_output = 0.0)`
Конструктор модели системы

`a` — коэффициент A модели

`b` — коэффициент B модели

initial_output — начальное выходное значение

`double update(double input)`
Обновляет состояние модели на один шаг и возвращает выход

`input` — входное воздействие (управляющий сигнал)

Возвращает: текущее выходное значение системы

`double updateWithNoise(double input)`
Обновляет состояние модели с добавлением шума измерений

`input` — входное воздействие

**Возвращает:** выходное значение с добавленным шумом

`void reset(double initial_output = 0.0)`
Сбрасывает состояние модели к начальному значению

`initial_output` — новое начальное значение

`void setNoiseLevel(double level)`
Устанавливает уровень шума для измерений

`level` — стандартное отклонение шума (0 для отключения)

`pid_controller_test.cpp`
**Тесты для проверки корректности работы ПИД-регулятора**
Набор unit-тестов для проверки функциональности класса PIDController.

Тестовые случаи:

`testPidConstructor()` — Проверяет корректность инициализации параметров

`testPidCalculation()` — Проверяет вычисление управляющего воздействия

`testPidReset()` — Проверяет сброс внутреннего состояния

`testPidWindupProtection()` — Проверяет защиту от насыщения интегральной составляющей

`testPidWithSystemModel()` — Интеграционный тест с моделью системы
